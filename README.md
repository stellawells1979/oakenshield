

    本项目组织了两个机器人，分别是搜索机器人，规则机器人，它们都有一个别名 search_bot,rules_bot
    以供开发者要维护更新时易于阅读，机器人详功能细和参数，在 bot 文档中描述和定义
    
    为方便阅读和维护，有必要规范一下描述惯例
    
    在类中被初始化的参数统称为类属性，在类中定义的功能函数统称为方法
    其它定义的功能模块统称为函数，对项目中的某些参数或参数集，统称为参数或对象

    项目中将需要向服务器发送的信息称为请求对象
    
    由于开发者水平太过于普通且身兼数职，开发过程将是一个断断续续的堆码工程，项目中的代码，逻辑很难形成连惯，缺少专业知识，
    他甚至无法将项目描述到自己能看懂，包括当前的描述文档，他都有可能遗漏了某些说明，但请相信，他一直在努力将所有的变得清晰
    明了，即使过一段时间他重新阅读这些代码时依然是一脸懵逼
    
    部分参数定义说明以下：

    callback_data: (str) 完全由开发者定义，本参数是用于消息键盘按钮的回调信息，开发者在定义键盘时必须为每个按钮根据
        某些逻辑定义并传递本参数，当用户点击了某个按钮时，服务器会原样返回这个参数，开发者可以根据此参数解析用户的操作
        意图，并据此做出响应，为此，你必须熟知此参数的结构和定义逻辑。telegram bot api 官方说明此参数长度不得超过60
        个字符，开发者应该尽量控制长度。在当前项目，本参数是由指定符号分隔成5部分的字符串，有两种应用场景

        如果应用于规则机器人使用正杠号（|）分隔，第一段字符指定了

        如果是应用于搜索机器人通常也是使用正杠号（|）分隔，但如果用户的搜索关键字包含了该符号，则会使用其它符号，为此你
        必须在编写代码时有明晰的处理逻辑和注释。

        不管哪种应用场景，此参数的每一段都不能为空，你可以用 0 代替无需定义的段落

    本项目约定函数参数的传递及接收规则：
        
        bot: 传递或接收一个机器人的别名
        bots： 传递或接收一个机器人的基本参数集和自定义的其它参数
        bot_infos: 传递或接收由多个机器人的基本参数和自定义的其它参数组成的参数集
